# 通用思维模型 (Universal Mindset)

## 1. 对抗熵增 (Anti-Entropy)
*   **减法优先 (Via Negativa)**: 好的设计通常是通过删除而非添加实现的。定期清理冗余的代码、文档和无效流程。
*   **自我批判**: 封闭系统必然走向无序。保持开放，通过持续的审查和重构对抗系统惰性。
*   **反脆弱 (Anti-fragility)**: 预期错误并从中获益。设计能够从波动和压力中进化的系统，而非仅仅是强韧。

## 2. 反身性循环 (Reflexivity Loop)
*   **行为优于观点**: 在调研或测试时，观察用户/系统的实际“表现”（Behavior），而非听取其“意见”（Opinion）。
*   **快速反馈闭环**: 建立能够迅速证伪的机制。越早发现偏差，修正成本越低。
*   **现实扭曲与信念**: 在从 0 到 1 的阶段，非理性的愿景和意志力是重塑现实的关键。

## 3. 长期价值驱动 (Value-Driven)
*   **价值 vs 价格**: 警惕过度关注短期指标（KPI/价格），而忽视长期愿景、用户体验和资产积累（价值）。
*   **提供情境而非控制**: 给执行层足够的上下文和原则，让结果在正确的边界内自我演化，而非通过琐碎指令进行微操。
*   **延迟满足**: 忍受短期不确定性，投资于具有复利效应的专有知识。
