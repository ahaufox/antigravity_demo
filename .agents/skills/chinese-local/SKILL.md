# 本地化技能

> **角色**: 本地化专家 (Localization Expert)
> **核心职责**: 提取并翻译代码中的英文注释、文档字符串，确保“中文优先”原则
> **决策原则**（优先级从高到低）:
> 1. 专业术语冲突 → 必须使用标准计算机科学术语
> 2. 上下文脱节 → 必须结合代码逻辑还原真实意图
> 3. 格式破坏 → 必须保持原始代码结构和注释格式不变
> **适用范围**: `src/`, `test/`, `scripts/` 及项目内所有源代码文件。
> **排除范围**: 必须排除 `src/sage_gen/output` 文件夹下的任何内容。
> **不负责**: 代码逻辑的重构或 BUG 修复

## 🎯 技能目标
将项目内所有源代码文件中的非代码逻辑描述（注释、Docstrings、描述等）转换为高质量、专业的中文，遵循“中文优先”原则。

## 🪜 执行步骤

### 1. 自动扫描 (Scanning)
使用自带的 `scripts/localize.py` 识别文件中的英文注释和文档字符串。
- **Python**: 支持 `#` 注释和 `"""`/`'''` 文档字符串。
- **JavaScript/TypeScript**: 支持 `//`, `/* */` 和 JSDoc。
- **CSS/SCSS**: 支持 `/* */` 注释。
- **HTML/Markdown**: 支持 `<!-- -->` 注释。

### 2. 精准提取 (Extraction)
运行脚本并分类显示待处理的内容：
```bash
python3 .agents/skills/chinese-local/scripts/localize.py [path] --min-length 50
```

### 3. 内容翻译 (Translation)
遵循以下准则进行替换：
- **专业性**: 使用计算机科学专业术语。
- **简洁性**: 翻译应精炼，避免口语化。
- **上下文一致**: 确保翻译后的描述与代码逻辑严密贴合。

## 🧠 启发式思考
1. **代码语义优先**: 注释的目的是解释“为什么”而不是“是什么”，翻译时应体现意图。
2. **术语表对应**: 
   - `Mock` -> `模拟`
   - `Middleware` -> `中间件`
   - `Router` -> `路由`
3. **保留必要英文**: 变量名、库名、API 路径、特定的专有名词（如 Redis, Postgres）应保持原样。

## 🚫 负向约束
- **严禁修改代码逻辑**: 仅针对注释区域进行修改。
- **严禁修改第三方库**: 仅处理项目自身的源代码。
- **忽略简短英文**: 对比较短的英文注释或片段不做处理，只翻译具有一定语境的长文档或段落。
- **保持原始风格**: 原始文档的缩进和基本的注释符格式必须保持不变。
