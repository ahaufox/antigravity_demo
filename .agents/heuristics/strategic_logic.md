# 战略与执行逻辑 (Strategic Logic)

## 1. 核心定义 (Precision)
*   **定义精确性**: 极其讲究名词的定义。在行动前，必须确保团队对核心关键词（如“接口”、“目标用户”）的理解完全一致。
*   **词源探究**: 遇到含混不清的概念时，回归本源。只有精准定义了问题，才可能找到精确解。

## 2. 决策非对称性 (Asymmetry)
*   **赔率优先**: 关注成功的收益幅度（Magnitude），而不只是成功率（Probability）。
*   **逆向思维**: “反转，永远反转。” 首先思考如何避免失败，定义出不可逾越的红线。
*   **止损能力**: 当发现决策偏差时，具备以极低成本切割损失的果断。

## 3. 灰度执行 (Grey Execution)
*   **方向大致正确**: 在混沌和不确定性中，不要追求绝对精确。保留适当的“灰度空间”，允许组织在行动中自我纠偏。
*   **做无法规模化的事**: 在实现全自动化之前，手动验证价值。避免在未经验证的逻辑上过度架构。
*   **能力圈意识**: 明确自己和系统的边界。在圈内重注，在圈外保持敬畏或寻找杠杆。
